Основная конфигурация
- bootstrap.servers: Список пар хост/порт для установки первоначального соединения с кластером Kafka.
- client.id: Необязательный идентификатор, помогающий отслеживать запросы.

Надежность сообщений
- acks: Управляет надежностью сообщений. 0 означает отсутствие подтверждения, 1 - подтверждение от лидера партиции, all - подтверждение от всех реплик в синхронном состоянии.
- retries: Количество попыток продюсера повторно отправить запись, если отправка не удалась.

Пакетная обработка и сжатие
- batch.size: Продюсер будет пытаться объединять записи в меньшее количество запросов, когда несколько записей отправляются в ту же партицию.
- compression.type: Тип сжатия, используемый для записей. Варианты: none, gzip, snappy, lz4 или zstd.

Разбиение на партиции
- partitioner.class: Класс разбивателя партиций, используемый для определения партиции для каждого сообщения.

- key.serializer: Класс сериализатора для ключей сообщений.
- value.serializer: Класс сериализатора для значений сообщений.

Расширенная конфигурация
- buffer.memory: Общий объем памяти в байтах, который продюсер может использовать для буферизации записей.
- max.in.flight.requests.per.connection: Максимальное количество неподтвержденных запросов, которые клиент отправит по одному соединению, прежде чем заблокируется.
- enable.idempotence: Обеспечивает, что продюсер может надежно повторять отправку без дублирования сообщений.
